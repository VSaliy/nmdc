$NickChange
« on: May 02, 2007, 09:16:01 PM »

Syntax:
Quote$NickChange <oldnick> <newnick>|
<oldnick>  the nickname the user currently uses
<newnick> the nickname the user is renamed to
If the users client signals NickChange in Supports, it will be send $ClientNick <newnick>.
This client will have to update it''s own nickname, $MyINFO string and userlist.
If the client does not support NickChange, the hub will apply these changes for the client on every data exchange.
To all other users, a $Quit <oldnick> will be send, plus a $MyINFO $ALL string of the user with new nick.
If the user is an Operator, $OpList will be updated apropriately as well, so the user will keep their status.
In case a client does not support NoHello, a full $OpList will be send as well as a $Hello <newnick>
NickChange support is designed by Lord_Zero and CrazyGuy and implemented in HexHub, security bots and scripts
