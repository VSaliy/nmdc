Reporting an error
« on: March 08, 2007, 02:10:36 PM »
To report an error, please include the following information (if possible):
- what conditions or circumstances caused that error
- debug information (in case of a crash)
To obtain debug information, you need to do the following:
Win95/98: click on "Details" in the "Illegal operation" dialog, and paste the information in your report.
Wint2k/XP: find the log file called drwtsn32.log - to do this, start drwtsn32.exe (that is located in System32 folder) and see the location where the log file is saved. By default, it is saved in a hidden directory, so searching for it may not give any results. In drwtsn32.log find the last crash report and add to your report the part that contains "FAULT ->".
If the procedure that caused the error is in HeXHub''s address space, in a short period of time a corrected version will be uploaded at sourceforge.
An "int 3" or "Fatal exception 3" found in HeXHub''s address space is usually caused by an error in a plugin. Unload all plugins and re-enable them each one at a time to see which plugin causes that error. If the error is caused by a plugin, or the error is found in a plugin''s address space, first thing you should do is check if there are any conflicts between that plugin and others. Move that plugin up in the list of plugins so it will the first in list. If this doesn''t solve your problem, select that plugin in the list and click "Settings". Uncheck all its rights and related events then check them back each one at a time to see which action or event causes that error in the plugin. Include the result you find in your error report to help us correcting it.
