Hello guys,
I noticed that the members of this forum seem to be quite interested in scripting hubs.
I have made PeerAware, an NMDC compatible client with support for scripting in the chat window.
I have no idea if you find this interesting, but as an example I made a hub script where you can play a card game after joining a hub. I also have examples of playing chess and battleship, all quite rough, but the purpose has been to demonstrate the concept. A screenshot is here: http://www.peeraware.com/PlayCardGame.jpg
Other than that it works much like dc++ except that it indexes contents of shared documents.
The script is written in html/javascript and are quite simple. You get to receive, transmit and broadcast messages from javascript, and do whatever you want based on those operations.
How the scripting works is that the hub sets up the script that clients should load, and all clients that connect automatically load the script upon joining. Incompatible clients only get normal chat.
Anyway, you might find it interesting, it is available here: http://www.peeraware.com/

I think you got it just about right:
On login to the hub you get a message from the hub: $HubScript url|
The client then loads this hubscript and displays that before accepting more input from the hub.
The Texas Hold'em poker hubscript is here: http://www.peeraware.com/hubscript/cardgame/hub.html
When you create a hub in peeraware you specify that this hubscript should be active in the hub. The hub will run as a service on the computer where it is installed.
Some simpler examples:
Play battleship in private chat only:
http://www.peeraware.com/hubscript/withbattleship/hub.html
Play chess for all to see in public hub:
http://www.peeraware.com/hubscript/withchess/hub.html
They are quite primitive, but still display how you can write p2p html/javascript apps on top of nmdc.
Any input is appreciated!
